Détails techniques

Le site web est entièrement statique et réalisé en HTML5 et CSS3 purs. Pas de BootStrap, de JavaScript, de PHP ou de préprocesseurs (LESS, SASS, ...).
Cependant, deux CDN ont été utilisés : celui de Google Fonts (pour la police) ainsi que celui de Font Awesome (pour les logos).
Le projet contient un total de 16 pages HTML ainsi qu'1 fichier CSS et des images.
Les pages HTML contenant « AddCart », « FullCart », « Empty », ... permettent de simuler une navigation cohérente en fonction de si vous avez ou non ajouté un élément à votre panier. Les liens redirigent donc vers ces pages nécessaires à la bonne navigation.
Seules les pages « Qui sommes nous ? » et « Détails techniques » ne sont pas prises en compte dans la navigation cohérente. Si vous cliquez dessus alors que vous aviez ajouté un produit à votre panier, cela ne se verra pas et votre panier sera vide.

Nous avons tenté de rendre le site un maxium accessible en choisissant les bonnes balises et en n'oubliant pas les attributs alt="" sur les images par exemple, on encore les label associés aux input ou des title="" sur les liens.
Le code CSS a été séparé du code HTML (mis à part pour 2 paragraphes dans la page aboutus.html car il s'agit de contenu exceptionnel associé au projet (nos noms et un disclaimer)).
Nous avons utilisé énormément de classes pour sélectionner nos balises en CSS, cependant nous n'utilisons pas de méthodologie de nommage (OOCSS, BEM, ...) bien que certaines classes peuvent y ressembler.

L'unité CSS que nous avons le plus utilisé est le rem. En effet, le rem se base sur une unité définie sur la balise html et ne dépend donc pas des balises parents tout en étant plus souple que le pixel. Ainsi, en ayant défini une unité de base de 16px (celle par défaut des navigateurs) via la propriété font, nous nous assurons de toujours avoir des tailles cohérentes (taille de la police, marges externes/internes, largeur, hauteur, ...).

Un slider animé et responsive a été mis en place ainsi que plusieurs effets de survols qui disparraissent en fonction de la taille de l'écran (notamment pour ne pas gêner la navigation sur écrans tactiles).

La méthode de mise en page utilisée est principalement le module natif de CSS3 : FlexBox. Nous l'avons énormément utilisé, surtout pour des alignements centrés (verticalement) de nos éléments.
Les espacements sont majoritairement gérés avec les marges internes/externes, à l'exception d'une div vide au niveau du header qui est utilisée afin d'assurer que le titre et la barre de recherche soient bien centrés grâce à un flex: 1; sur les éléments enfants du header.
Les propriétés positions ont été utilisées pour afficher un menu fixe, initialement appartenant à la section du slider, en bas de l'écran sur les écrans des téléphones portables.

Toutes les pages HTML et CSS ont été vérifiées avec le validateur de la W3C et aucune erreur ne semble être survenue.

La plupart des navigateurs semblent être supportés (même Internet Explorer 11 !) bien que nous n'ayons pu tester avec une version récente de Safari (pas de Mac). Pour le responsive, FlexBox a été utilisé pour replacer les éléments (wrap). Ainsi que l'utilisation en amont de propriétés responsives (rem pour le zoomage, max-width plutôt que width sur les éléments de type block, ...) et enfin les media queries.
